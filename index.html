Basic rules:

  1. Create your maze

    • Click each field to set it state
      • wall
      • path
      • start
      • stop
    • there can be only one start and stop
    • We can't make a path's on labyrinth's border
    • Can't set start/stop on corners

  2. Configure algorithm

    • Select algorithm
    • Turn on / off graph optimisation - responsible for selecting only significant fields
    • Select solve direction - the way of selecting next field

  3. Run algorithm

    • Run with auto mode and selected delay between steps
    • Run with manual mode and click on the maze to take next step
    • You can't edit maze and algorithm while runing - stop it and clear solution

Algorithm

Breadth-first search

BFS algorithm traverses a graph in a breadthward motion and uses a queue to remember to get the next vertex to start a search, when a dead end occurs in any iteration.

More info:

wikipedia.org/wiki/Breadth-first_search

Deep-first search

DFS algorithm is a recursive algorithm that uses the idea of backtracking. It involves exhaustive searches of all the nodes by going ahead, if possible, else by backtracking.

More info:

wikipedia.org/wiki/Depth-first_search

A*-ish

A* is an informed search algorithm, or a best-first search, meaning that it is formulated in terms of weighted graphs: starting from a specific starting node of a graph, it aims to find a path to the given goal node having the smallest cost (least distance travelled, shortest time, etc.). It does this by maintaining a tree of paths originating at the start node and extending those paths one edge at a time until its termination criterion is satisfied.

More info: wikipedia.org/wiki/A*_search_algorithm
